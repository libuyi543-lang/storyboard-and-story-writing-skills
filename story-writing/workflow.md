# Story Development Workflow

## 1. Start With a Premise, Not a Full Plot

Capture:

- protagonist
- situation
- desire or problem
- destabilizing element

Then write a logline.

Template:

`When ___ happens, a ___ must ___ before/otherwise ___ .`

## 2. Define the Core Question

Write:

`Can the protagonist ___ ?`

Then answer these:

- Event: what concrete result are we following?
- Emotion: which bond, wound, fear, or longing deepens the story?
- Value: what human idea is the story arguing, testing, or discovering?

If only the event layer is clear, keep developing.

## 3. Test Whether the Story Can Stand

Ask five questions:

1. Is the protagonist clear enough?
2. Is there a strong enough goal, danger, or situation?
3. Is the obstacle real and escalating?
4. Does the setup contain turn potential?
5. Can the story end with a meaningful change or release?

If any answer is weak, repair that before expanding.

## 4. Build the Large Shape

Use one of these:

- Three-act:
  setup,
  escalation,
  climax-resolution
- Five beats:
  catalyst,
  complications,
  reversal,
  climax,
  release
- 起承转合:
  opening state,
  development,
  sharp turn,
  settling outcome

Pick one model and stay consistent while outlining.

## 5. Link Character and Structure

For the protagonist, define:

- outer goal
- emotional need
- flaw, blind spot, or contradiction
- possible value-level lesson

Then make sure major turns hit both plot and character.

Bad version:
the plot changes, but the protagonist's inner struggle is untouched.

Better version:
the plot change pressures the protagonist's weakness or belief.

## 6. Expand Into Beat Sheet

Turn the structure into a sequence of beats.

For each beat, write:

- what happens
- what changes
- why the audience keeps watching

If a beat does not increase curiosity, pressure, or emotion, compress or remove it.

## 7. Turn Beats Into Scene List

For each scene, write:

- scene purpose
- protagonist tactic
- opposing force
- turn or result
- next expectation created

Default test:
If the scene vanished, would the audience lose necessary change? If not, cut or merge it.

## 8. Design Reversals and Setbacks

Use reversals when the story starts moving in a straight, predictable line.

A good reversal:

- changes expectation
- costs the protagonist something
- forces a new tactic
- deepens either emotion or theme

## 9. Handle Climax

The climax is usually the hardest part because it must pay off:

- the event question
- the emotional question
- the value question

Check all three.

If the climax only solves the event layer, it often feels empty.

## 10. End With Release

The ending should show the after-state.

Possible release forms:

- success with cost
- failure with meaning
- relationship repaired or broken
- a belief proven or overturned

The audience should feel what changed, not just be told that it changed.

## 11. Multi-Protagonist or Multi-Line Stories

When handling multiple leads or lines:

- give each line a clear local question
- make sure lines differ in dramatic function
- avoid parallel lines that feel interchangeable
- know where lines cross, echo, or collide

If one line can disappear with little loss, it is probably underbuilt.

## 12. Idea Development

When the user has only a seed idea:

1. state what is interesting about the seed
2. derive protagonist + problem
3. derive a central question
4. find a contradiction or emotional wound
5. search for the sharpest turn
6. build structure from there
